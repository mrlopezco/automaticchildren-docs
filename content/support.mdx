# Support

For bugs, feature requests, and questions about the Automatic Children extension, here's how to get help.

## Report a bug

If something isn't working as expected, please include:

- A short description of what you were trying to do.
- The type of work item you were using (e.g., User Story, Bug, Epic).
- Whether the problem is with a **general rule**, a **cascade**, or the **settings page**.
- Any error message shown by the extension or in the browser console (if safe to share).

The more detail you can provide, the faster we can look into it.

## Request a feature

Have an idea to make Automatic Children better? It helps to describe:

- The problem or workflow you're trying to improve.
- How you currently work around it.
- Any examples of general rules or cascades you'd like to be able to create.

## How to reach us

- Use the **contact** or **marketplace** link from the **About** tab in **Project Settings > Automatic Children**.
- Or visit the [Visual Studio Marketplace listing](https://marketplace.visualstudio.com/items?itemName=LopezColl.automatic-children) for the extension's contact details and issue tracker.

## About the extension

Automatic Children is an Azure DevOps extension designed to make repeatable project structures easier and more reliable. The goal is to give teams a lightweight, no-code rules engine that stays close to native Azure DevOps concepts — without introducing additional services or infrastructure.

For version history and updates, check the **About** tab in Project Settings > Automatic Children, or visit the marketplace listing.

If you find the extension useful, a rating or review on the marketplace goes a long way in helping other teams find it.

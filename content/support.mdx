# Support

For bugs, feature requests, and more information about the Automatic Children extension, use the resources below.

## Report bugs

If you encounter unexpected behaviour, please include:

- A short description of what you were trying to do.
- The type of work item you were using (e.g. User Story, Bug, Task).
- Whether the problem is with **rules**, **cascades**, or the **settings** page.
- Any error message shown by the extension or in the browser console (if safe to share).

## Feature requests

For ideas to improve Automatic Child Items, it helps to describe:

- The problem or workflow you are trying to improve.
- How you currently work around it without the extension.
- Any examples of rules or cascades you would like to be able to configure.

## How to reach the maintainer

- Use the **contact** or **marketplace** link from the **About** tab in **Project Settings > Automatic Children**.
- Or use the repository’s issue tracker / contact details from the [Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=LopezColl.automatic-children) listing.

Include as much detail as is safe to share from your environment.

## About the extension

Automatic Child Items is maintained as an Azure DevOps extension to make repeatable project structures easier and more reliable. The goal is to give teams a lightweight rules engine that stays close to native Azure DevOps concepts, without introducing additional services or infrastructure.

For version history and updates:

- Open the **About** tab in **Project Settings > Automatic Children**.
- Follow the link to the Visual Studio Marketplace listing.

If you find the extension helpful, consider sharing feedback or a rating on the marketplace to help guide future improvements.
